키워드 추출 흐름

BQ SQL → ML.GENERATE_TEXT → keyword_results

1
Remote Model 등록 (최초 1회)
CREATE MODEL로 Vertex AI Gemini를 BigQuery Remote Model로 연결
2
ML.GENERATE_TEXT 실행
SQL 1줄로 미처리 리뷰에 Gemini 프롬프트 전송
prompt에 "키워드와 감성 분류를 JSON으로 반환해줘" 지시
3
JSON 응답 파싱
JSON_VALUE / JSON_QUERY_ARRAY로 word, sentiment 추출
4
keyword_results 테이블 INSERT
word · sentiment · shop_id · week 컬럼으로 결과 저장
5
처리 완료 플래그 업데이트
UPDATE reviews SET is_processed = TRUE로 재처리 방지
📝

AI 요약 흐름

BQ SQL → ML.GENERATE_TEXT → summary_results

1
주차별 리뷰 집계 쿼리
STRING_AGG로 같은 가게·주차의 리뷰 본문을 하나로 연결
2
ML.GENERATE_TEXT 요약 요청
BQ SQL 안에서 Gemini에 "긍정 요약 / 부정 요약을 JSON으로" 지시
3
JSON 응답 파싱
JSON_VALUE(ml_generate_text_result, '$.positive') 등으로 분리 추출
4
summary_results 테이블 INSERT
positive_summary · negative_summary · shop_id · week 저장
5
처리 완료 플래그 업데이트
UPDATE reviews SET is_summarized = TRUE로 재처리 방지

Python 없이 SQL 한 줄로 AI

ML.GENERATE_TEXT 함수 하나로 BigQuery SQL 안에서 Gemini 직접 호출.
별도 Python 서버·API 관리 없이 데이터 파이프라인을 SQL로만 완결.

🔄

스케줄 싱킹으로 역할 분리

is_processed 플래그로 BQ 처리 여부 추적.
배치가 변경분만 감지해 App DB에 upsert → 중복 처리 없음.

⚡

App은 읽기 전용

Application은 BigQuery에 직접 쿼리하지 않음.
싱킹된 App DB만 사용해 응답 속도·비용·부하 모두 최적화.

BigQuery ML 실제 SQL
Python 없이 SQL 안에서 Gemini 호출 — Remote Model 등록 → ML.GENERATE_TEXT → JSON 파싱 → INSERT
bigquery_ml_keyword_summary.sql
-- ① Vertex AI Gemini를 BigQuery Remote Model로 등록 (최초 1회) CREATE OR REPLACE MODEL `project.dataset.gemini_model` REMOTE WITH CONNECTION `project.region.vertex-connection` OPTIONS (endpoint = 'gemini-1.5-flash'); -- ② 키워드 추출: 미처리 리뷰에 ML.GENERATE_TEXT 실행 INSERT INTO `project.dataset.keyword_results` (word, sentiment, shop_id, week) WITH ai_response AS ( SELECT r.shop_id, r.week, ML.GENERATE_TEXT( MODEL `project.dataset.gemini_model`, (SELECT CONCAT( '다음 리뷰에서 핵심 키워드와 감성(positive/negative/neutral)을 ', 'JSON 배열 [{word, sentiment}] 형식으로만 반환하세요.\n리뷰: ', r.content ) AS prompt), STRUCT(0.1 AS temperature, 512 AS max_output_tokens) ) AS result FROM `project.dataset.reviews` r WHERE r.is_processed = FALSE ) SELECT JSON_VALUE(kw, '$.word') AS word, JSON_VALUE(kw, '$.sentiment') AS sentiment, ar.shop_id, ar.week FROM ai_response ar, UNNEST(JSON_QUERY_ARRAY( JSON_VALUE(ar.result.ml_generate_text_result, '$.candidates[0].content.parts[0].text') )) AS kw; -- ③ 요약 생성: 주차별 리뷰 묶어서 긍/부정 요약 동시 추출 INSERT INTO `project.dataset.summary_results` (shop_id, week, positive_summary, negative_summary) WITH weekly_reviews AS ( SELECT shop_id, week, STRING_AGG(content, '\n') AS all_reviews FROM `project.dataset.reviews` GROUP BY shop_id, week ), ai_summary AS ( SELECT shop_id, week, ML.GENERATE_TEXT( MODEL `project.dataset.gemini_model`, (SELECT CONCAT( '다음 리뷰들의 긍정 포인트와 부정 포인트를 각각 한 문장으로 요약해주세요. ', '{"positive": "...", "negative": "..."} JSON으로만 반환하세요.\n리뷰:\n', wr.all_reviews ) AS prompt), STRUCT(0.3 AS temperature) ) AS result FROM weekly_reviews wr ) SELECT shop_id, week, JSON_VALUE(JSON_VALUE(result.ml_generate_text_result, '$.candidates[0].content.parts[0].text'), '$.positive') AS positive_summary, JSON_VALUE(JSON_VALUE(result.ml_generate_text_result, '$.candidates[0].content.parts[0].text'), '$.negative') AS negative_summary FROM ai_summary; -- ④ 처리 완료 마킹 UPDATE `project.dataset.reviews` SET is_processed = TRUE WHERE is_processed = FALSE;

# 슬라이드 2 / 6 — AI 분석 파이프라인

## 타이틀

**7단계 하이브리드 분석 파이프라인**

로컬 처리 + 최소 LLM 호출로 정확도와 비용을 동시에 잡는다

---

## 파이프라인 플로우

```
리뷰 원문 (JSON)
    │
    ▼
[Phase 0] 기간 분류                  ← 비용 $0  (로컬)
    │  → 월별 / 주별 버킷 분배
    │  → ISO 8601 기준 주차 계산
    │
    ▼
[Phase 1] kiwipiepy 형태소 분석      ← 비용 $0  (로컬)
    │  → 명사/형용사 추출
    │  → 키워드 빈도 집계
    │  → 룰 기반 감성 태깅 (긍정/부정/중립)
    │
    ▼
[Phase 2] 애매한 케이스 필터링       ← 비용 $0  (로컬)
    │  → 이중부정 / 혼합 감성 / 미등록어
    │  → 평점-감성 불일치 감지
    │  → 전체의 약 15~25%만 선별
    │
    ▼  ← 여기서부터 LLM 호출
[Phase 3] LLM 호출 (최소 토큰)      ← 비용 최소 (~200토큰)
    │  ├─ (a) 애매한 리뷰 감성 판단
    │  └─ (b) 통계 → 자연어 리포트 생성
    │
    ▼
[Phase 4] 결과 병합                  ← 비용 $0  (로컬)
    │  → 로컬 분석 + LLM 판정 통합
    │
    ▼
[Phase 5] 별점 트렌드 계산          ← 비용 $0  (로컬)
    │  → 주별 별점 추이
    │  → 이동평균, 변동률
    │  → 카테고리별 별점 산출
    │
    ▼
[Phase 6] 비교 분석                  ← 비용 $0  (로컬)
    │  → 전월/전주 대비 변화
    │  → 급변 감지 알림
    │
    ▼
 출력 (JSON + 차트 데이터)
```

---

## 핵심 설계 원칙

| 원칙 | 내용 |
|---|---|
| 비용 최소화 | 75~85%는 로컬 처리, LLM은 필요한 경우만 |
| 정확도 확보 | 애매한 케이스는 LLM이 정확하게 판단 |
| 확장 가능 | 각 Phase는 독립 모듈로 교체/업그레이드 가능 |

---

## 감성 분류 예시

| 리뷰 | 분류 방식 | 결과 |
|---|---|---|
| "맛있어요! 또 올게요" | 로컬 룰 기반 | 긍정 |
| "별로였어요" | 로컬 룰 기반 | 부정 |
| "나쁘지 않은데 굳이 또 시킬 정도는 아니에요" | LLM 판단 | 중립/부정 |
| "가격 대비 괜찮" | LLM 판단 | 중립 |

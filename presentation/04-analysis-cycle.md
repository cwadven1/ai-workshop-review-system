# 슬라이드 4 / 6 — 분석 주기

## 타이틀

**월별 종합 + 주별 실시간 모니터링**

ISO 8601 기준 이중 분석 주기

---

## 두 가지 분석 주기

### 월별 분석
- 1월 ~ 12월, 월 마감 후 해당 월 전체 리뷰 **확정 분석**
- 목적: 종합 성과 평가, 전월/전년 동월 비교
- 리뷰 수가 많아 높은 신뢰도

### 주별 분석
- ISO 8601 기준, **월요일 시작**
- 진행 중인 달의 실시간 추이 모니터링
- 급변 감지 → 즉시 알림

---

## 분석 흐름

```
[3월 월별 분석] ← 3월 1일~31일 전체 리뷰 확정
       │
       ▼ 3월 마감
[4월 주별 분석]
  ├─ W13: 3/30 ~ 4/5    ← 평점 3.0 (상승 시작)
  ├─ W14: 4/6 ~ 4/12    ← 평점 3.5 (지속 상승)
  ├─ W15: 4/13 ~ 4/19   ← 평점 4.5 (리뉴얼 효과)
  └─ W16: 4/20 ~ 4/26   ← 평점 5.0 (최고점 달성)
       │
       ▼ 4월 마감
[4월 월별 분석] ← 4월 전체 확정
```

---

## 설계 결정사항

| 항목 | 결정 | 이유 |
|---|---|---|
| 주차 기준 | 월요일 시작 | ISO 8601 표준, 영업일 기준 자연스러움 |
| 분석 타이밍 | 배치 (수동 트리거) | 프로토타입 단계, 추후 cron 추가 가능 |
| 월/주 겹침 처리 | 더 많은 날이 속한 월로 귀속 | 4/28~5/4 → 4월 5주차 (4일 > 3일) |
| AI 총평 원칙 | 원인 분석만, 통계 수치 배제 | 수치는 차트로, 총평은 "왜" 변했는지에 집중 |
| 최근 분석 기간 | 최근 4주 (W13~W16) | 단기 트렌드 파악에 최적 |

## 실제 데이터 예시: 파스타 팩토리

| 주차 | 평균 평점 | 리뷰 수 | 주요 키워드 | 트렌드 |
|---|---|---|---|---|
| 2026-W13 | 3.0 | 3건 | 보통, 무난한, 평범한 | 기준 |
| 2026-W14 | 3.5 | 2건 | 보통, 무난한, 평범한 | ↑ 상승 |
| 2026-W15 | 4.5 | 4건 | 리뉴얼, 맛집, 크림소스 | ↑↑ 상승 |
| 2026-W16 | 5.0 | 3건 | 바삭한, 알덴테, 리뉴얼 | ↑↑↑ 최고점 |

**→ W15부터 "리뉴얼" 키워드 등장 → 품질 개선 효과 확인 가능**

## WeeklySummary 데이터 모델

```python
class WeeklySummary(models.Model):
    store = ForeignKey(Store)
    year_week = CharField()          # "2026-W16"
    review_count = IntegerField()
    avg_rating = DecimalField()
    rating_change = DecimalField()   # 전주 대비 변화
    sentiment_distribution = JSONField()  # {"positive": 3, "neutral": 0, "negative": 0}
    top_keywords = JSONField()       # [{"keyword": "리뉴얼", "count": 3}]
    summary = TextField()            # AI 생성 주별 요약
    highlights = JSONField()         # {"good_points": [...], "bad_points": [...]}
```

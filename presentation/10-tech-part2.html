PART 2 · 기술 구현
어디가좋은데?

주차별 요약 AI 생성

주차 통계 수집 → GPT-4o-mini 프롬프트 → ShopWeekReview.summary + ShopWeekSentimentReveiw(sentiment별) 저장

AI가 채우는 필드

stores_shopweekreview
summary

해당 주차 전체 리뷰를 종합한 AI 요약문.
평점·분위기·주요 반응을 3–4문장으로 서술.

stores_shopweekreviewinfo
sentiment + content

감성(positive / negative) 별로 각 1건씩 저장.
sentiment enum으로 구분, content에 해당 감성 패턴 분석 텍스트.

처리 파이프라인

STEP 1 · 수집

ShopWeekReview 조회

ShopWeekReview.objects.filter(shop=shop, year=year, week_number=week)
해당 가게·주차의 집계 레코드 조회. 데이터 없으면 조기 종료.

STEP 2 · 통계 준비 (로컬 · 비용 $0)

감성 수치 + 상위 키워드 수집

ShopWeekReview — positive_count / negative_count / neutral_count, average, count
ShopWeekReviewKeyword — count 기준 상위 N개 키워드 조회

STEP 3 · LLM

Gemini 2.0 Flash 요약 생성

실제 리뷰 텍스트 샘플(최대 15건) + 상위 키워드를 프롬프트에 삽입.
수치 재서술 절대 금지 — 리뷰에서만 발견할 수 있는 패턴·맥락 분석 요구.
JSON 형식으로 summary + positive_content + negative_content 반환.
API 키 없으면 키워드 기반 더미 텍스트로 자동 폴백.

STEP 4 · 저장

summary / content 저장 (이력 누적)

ShopWeekReview.update_or_create → summary 저장
ShopWeekSentimentReveiw.create(sentiment="positive") — 긍정 분석 텍스트
ShopWeekSentimentReveiw.create(sentiment="negative") — 부정 분석 텍스트
재계산 시 신규 레코드 추가, 이전 이력은 자동 보존.

GPT 프롬프트 템플릿

generate_summaries.py — _generate_with_gemini() 
# --- SYSTEM ---
당신은 음식점 리뷰 분석 전문가입니다.
반드시 아래 제공된 실제 리뷰 텍스트를 직접 읽고 분석하세요.
절대 금지: 별점·건수·긍정/부정/중립 비율 등 수치를 그대로 서술하는 것.
화면에 이미 표시된 통계 수치를 단순 반복하지 마세요.
대신: 리뷰 텍스트에서만 발견할 수 있는 패턴, 반복되는 맥락, 고객 경험의 본질을 분석하세요.

# --- USER ---
가게: {shop_name} ({year}년 {week_number}주차)

[실제 리뷰 샘플]
- {review_text_1}
- {review_text_2}
- ...  (최대 15건, 긍정 10 + 부정 5)

[상위 키워드]
- 긍정: {positive_keywords}
- 부정: {negative_keywords}

아래 JSON 형식으로만 응답해주세요.
positive_contents 와 negative_contents 는 빈 리스트 값이 될 수 있습니다.
{
  "summary":           "이 가게만의 특색과 고객이 반복 언급하는 핵심 경험 3문장. 수치 언급 절대 금지.",
  "positive_contents": ["고객이 칭찬하는 구체적 이유와 맥락1", "고객이 칭찬하는 구체적 이유와 맥락2"],
  "negative_contents": ["고객 불만의 구체적 맥락1", "반복 요청되는 개선 포인트2"]
}

ShopWeekReview — AI 저장 필드

🗂 stores_shopweekreview
타입 필드명 타입 설명
PK id AutoField 기본키
FK shop ForeignKey Store 참조 · unique_together(shop, year, week_number)
year IntegerField 집계 연도
week_number IntegerField ISO 주차 번호 (1–53)
count IntegerField 해당 주 리뷰 수 — 스케줄러 집계
average FloatField 해당 주 평균 별점 — 스케줄러 집계
positive_count IntegerField 긍정 리뷰 수 — 키워드 sentiment 기준 집계
negative_count IntegerField 부정 리뷰 수 — 키워드 sentiment 기준 집계
neutral_count IntegerField 중립 리뷰 수 — 키워드 sentiment 기준 집계
AI summary TextField GPT 생성 — 이번 주 전체 리뷰 종합 요약 (3–4문장) ★
created_at DateTimeField auto_now_add
updated_at DateTimeField auto_now — 증분 업데이트 watermark

ShopWeekSentimentReveiw — AI 저장 필드

🗂 stores_shopweekreviewinfo
타입 필드명 타입 설명
PK id AutoField 기본키
FK shop_week_review ForeignKey ShopWeekReview 참조 (on_delete=CASCADE) — 이력 누적
sentiment CharField 감성 구분 — positive / negative / neutral (enum)
AI content TextField GPT 생성 — 해당 sentiment 리뷰 패턴 분석 텍스트 ★
created_at DateTimeField 생성 시각 — ShopWeekReview.updated_at 과 매칭해 버전 식별
updated_at DateTimeField auto_now

AI 생성 특징

📋

리뷰 텍스트 샘플 포함

실제 리뷰 텍스트 최대 15건을 프롬프트에 삽입.
LLM이 수치 재서술 대신 맥락·패턴을 분석할 수 있도록 원문 제공.

🔄

폴백 전략

OpenAI API 키 없거나 오류 시 통계 기반 더미 텍스트 자동 생성.
개발·스테이징 환경에서 API 비용 없이 동작.

⚡

증분 실행

스케줄러가 ShopWeekReview.updated_at watermark 기준으로 변경된 주차만 AI 재생성.
변경 없으면 LLM 호출 0건.

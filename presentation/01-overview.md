# 슬라이드 1 / 6 — 프로젝트 개요

## 타이틀

**ReviewTime**
AI 기반 리뷰 분석 플랫폼

> 리뷰 100개를 읽는 데 걸리는 시간? — **3초**

---

## 문제 정의

| 현재 상황 | 문제점 |
|---|---|
| 배달 앱에 리뷰가 쌓인다 | 사장님도, 소비자도 다 읽을 수 없다 |
| 별점만 보고 판단 | "4.0점이어도 최근 2주 동안 계속 떨어지고 있다면?" |
| 리뷰 텍스트는 비정형 | 음식/배달/서비스 중 어디가 문제인지 모른다 |

---

## 솔루션

ReviewTime은 3가지 핵심 기술로 이 문제를 해결합니다.

### 1. 형태소 분석 (kiwipiepy)
- 로컬 실행, API 비용 **$0**
- 키워드 추출, 감성 태깅 자동화

### 2. LLM 하이브리드 전략
- 애매한 케이스 15~25%만 LLM 전달
- 토큰 사용량 **5,000 → 200 (25배 절약)**

### 3. 주별 트렌드 시각화
- 별점을 주식 차트처럼 시가/종가/고가/저가로 표현
- 급변 알림으로 위기 신호 조기 감지

---

## 기술 스택

- **백엔드**: Django (Python)
- **형태소 분석**: kiwipiepy
- **LLM**: Claude API (최소 토큰 사용)
- **프론트엔드**: Bootstrap 5 + Vanilla JS
- **데이터베이스**: SQLite

---

## 핵심 지표

| 지표 | 수치 |
|---|---|
| 분석 가능 리뷰 수 | 100건/가게 |
| LLM 토큰 절약 | 25배 |

| 카테고리 분류 | 음식/배달/서비스/가격/위생 |
